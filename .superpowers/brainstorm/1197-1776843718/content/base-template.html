Base Template — 2 Pendekatan

Section 2 of 4 — Shared Logic Pattern

A

Composable useInvitationTemplate()

Extract semua logic dari NusantaraTemplate ke sebuah composable. Tiap template baru tinggal panggil satu baris:

// JasmineTemplate.vue
const { groomName, brideName, primary,
  sectionEnabled, events, galleries,
  opened, openInvitation, showToast
} = useInvitationTemplate(props)

// <template> 100% custom visual HTML

Pros

  • Logic 1 tempat, 25+ template shared
  • Bug fix logic = fix semua template sekaligus
  • Template file = visual only, ringan

Cons

  • Perlu refactor NusantaraTemplate dulu
  • ~30-60 menit setup awal
B

Scaffold copy NusantaraTemplate

Script copy NusantaraTemplate.vue → NewTemplate.vue. Konverter edit bagian visual, logic sudah ada.

$ npm run make:template jasmine
→ copy NusantaraTemplate.vue
→ rename ke JasmineTemplate.vue
→ register di registry.js

Developer tinggal edit bagian <template>

Pros

  • Zero setup — langsung kerja
  • Setiap template self-contained
  • Konverter lihat contoh nyata di file yang sama

Cons

  • Logic terduplikasi di 25+ file
  • Bug fix harus update semua template
  • File besar per template
Rekomendasi: A (Composable)
Untuk 25+ template, logic terduplikasi = masalah jangka panjang. Composable = satu bug fix, semua template fixed. NusantaraTemplate tetap jadi referensi/contoh terbaik.
